[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture of the system.
First, the request enters the Security Checkpoint, where strict rules block sensitive data leakage by sanitizing inputs and telemetry spans.
Next, the request hits the RootAgent, which serves as our main ReAct orchestrator and decides how to route queries.
If the query requires weather info, the RootAgent invokes the get_weather tool, which acts as a simulated MCP tool to return weather states.
If the query needs localized clock info, the RootAgent routes it to the get_current_time tool to compute timezone offsets.
Finally, in developer mode, a Human ✋ approval node pauses execution to let operators inspect decisions, and all interactions are exported via OTel trace clients.
